Experiment 3 - Alternate Worlds



Description

This experiment showcases my work in using a tileset to generate two different maps: an overworld map and a dungeon map. Tileset Credit: Micro Tileset - Overworld & Dungeon by thkaspar
The overworld map features ice, snowy trees, snowy houses, and a body of water which is inspired by my personal thought of what a snowy town would look like.
The dungeon map features walls, rooms, and chests which is inspired by old-school dungeon rooms in video games.

Technical

  • Grids: Clicking the reseed button will generate a new grid based on a seed. This grid is parsed and the tiles will be placed based on the grid text.
  • Autotiling: For both of the grids, I use drawContext() and const lookup to properly draw the tiles corners depending on its neighbor.
  • Editing: The user can type certain codes to generate their own map.
    • For the overworld map, use w for water, h for house, : for trees, and . for ice.
    • For the dungeon map, use # for walls, . for room generation, and c for chests.
  • Incidentals: Different seed values lead to different tiles being used for my water, houses, trees, ice, walls, and chests because they utilize the random() function to select from the tileset.
  • Life: As time passes, the water fluctuates as if schools of fish are swimming by for the overworld map and the walls mimic silverfish travelling through the walls for the dungeon map.

    • Reflection

    Working with p5.js this time was a lot more difficult because I had to take time to understand how tilesets worked. I found it difficult to properly place the correct tile in the right spot, but once I understood this, all I needed to do was create ways to generate my maps. The grid text generation was difficult because I found myself trying out a lot of random math functions in conjunction with noise() until I got my desired outcome. I used a lot of ChatGPT to help my understanding of what values were being generated from noise() and this helped me adjust value parameters according to what I needed. After this, I had to integrate both of my projects into the same html, but I found "instance mode" to be easy to learn and following the provided guide made my website cleaner.
